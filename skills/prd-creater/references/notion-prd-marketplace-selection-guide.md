# Notion PRD Marketplace Selection Guide

Use this file after checking the synced catalog. It is a shortcut for choosing the closest template family before drafting.

## Fast filters
- `Official Notion`: Use when the user wants the safest or most canonical Notion baseline.
- `Free`: Use when the user wants templates that can be duplicated without purchase.
- `Paid`: Use when the user asks for premium or specialist community patterns.
- `Community`: Use when the user wants niche formats, creator-specific frameworks, or more opinionated structures.

## Template families

### 1. General PRD baseline
Use for standard product features, cross-functional alignment, or first-pass requirements docs.

Signals:
- Keywords like `PRD`, `Product Requirements Doc`, `Product Requirement Document`
- Need for context, goals, requirements, dependencies, and launch criteria

### 2. Internal tool and operations PRD
Use for back-office tools, admin workflows, internal dashboards, approval systems, or internal process changes.

Signals:
- Keywords like `Internal Tool PRD`, `Project Requests`, `Approvals`, `Business Requirements`
- Audience is ops, finance, support, or internal teams

### 3. Startup, MVP, and early-stage product docs
Use for lean scope, hypothesis-led launches, and high-speed shipping.

Signals:
- Keywords like `Startup`, `MVP`, `1-Pager`, `Opportunity Solution Tree`
- Need to emphasize assumptions, learning goals, and tight scope

### 4. AI and experimentation PRD
Use for AI features, growth experiments, structured learning plans, or LLM product ideas.

Signals:
- Keywords like `AI PRD`, `Structured AI PRD`, `Growth Experiment`
- Need for hypotheses, guardrails, and success metrics

### 5. Product spec and design-adjacent docs
Use when design, UX, or product spec detail is the center of gravity.

Signals:
- Keywords like `Product Spec`, `Design Spec`, `Design RFC`, `Case Study`
- Need for richer flows, edge cases, design rationale, or review checkpoints

### 6. Roadmap, prioritization, and intake
Use when the request is tied to prioritization, requests, scoring, or roadmap planning.

Signals:
- Keywords like `Quarterly Roadmap`, `RICE`, `Weighted Scoring`, `Feature Requests`
- Need for business value ranking, backlog triage, or prioritization logic

### 7. Technical, analytics, or engineering specs
Use for developer-facing features, analytics instrumentation, technical planning, or software specs.

Signals:
- Keywords like `Software Spec`, `Developer Portal`, `Tracking Plan`, `Architecture Strategy`
- Need for data definitions, technical constraints, or engineering handoff detail

## Default selection rule
1. Match the user's use case to the closest family.
2. Prefer official Notion templates if multiple options are equivalent.
3. Prefer free templates unless the user explicitly asks for premium community patterns.
4. If no exact match exists, blend the official Notion PRD spine with the closest family from the catalog.
